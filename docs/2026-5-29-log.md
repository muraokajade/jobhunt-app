## 2026-05-30

### 目的

App.tsx が肥大化してきたため、一覧表示テーブルを CompanyTable コンポーネントに分割する。

### 分割時の考え方

まず「どの JSX を App.tsx から消したいか」を決める。

今回消す対象は、企業一覧テーブル全体。

### 手順

1. App.tsx から消したい JSX 範囲を決める
2. その JSX 内で使っている state / 関数 / 定数を洗い出す
3. 子コンポーネントに渡す props を決める
4. Props の type を定義する
5. 子コンポーネント側で props を受け取る
6. App.tsx 側ではコンポーネントを呼び出すだけにする
7. 動作確認する

### 今回 CompanyTable に渡すもの

- companies
- loading
- priorityOptions
- statusOptions
- onPriorityChange
- onStatusChange
- onOpenDetail
- onDelete

### 学び

親コンポーネントである App.tsx は state と API 処理を持つ。

子コンポーネントである CompanyTable は、表示とユーザー操作を担当する。

つまり以下の役割分担にする。

App.tsx = 状態管理・API 通信  
CompanyTable = 一覧表示・操作イベント通知

### 注意点

table の tr 直下には td / th しか置かない。

select を直接 tr の中に置くと、列ズレの原因になる。

正しい形は以下。

```tsx
<tr>
  <td>
    <select>...</select>
  </td>
</tr>
```
