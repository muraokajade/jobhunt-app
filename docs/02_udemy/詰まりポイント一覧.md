こういう箇所は、教材で丁寧に説明すると価値が出る。

# 3. Laravel特有のメソッド・PHP組み込み関数・React側で必ず説明するもの

## Laravel / PHP側

### `Route::apiResource()`

`Route::apiResource()` は、API用のCRUDルートをまとめて作るLaravelのメソッド。

```php
Route::apiResource('companies', CompanyController::class);
```

これにより、以下のようなルートが自動で作られる。

```txt
GET /companies
POST /companies
GET /companies/{company}
PUT/PATCH /companies/{company}
DELETE /companies/{company}
```

注意点として、`/companies/dashboard` のような固定ルートは、`apiResource` より先に書く必要がある。

---

### `response()->json()`

`response()->json()` は、配列やデータをJSONレスポンスとして返すLaravelの関数。

```php
return response()->json([
    'summary' => [
        'total' => $companies->count(),
    ],
]);
```

React側は、このJSONを受け取って画面に表示する。

---

### `FormRequest`

`FormRequest` は、リクエストの入力値を検証するLaravelの仕組み。

例：

```php
StoreCompanyRequest
UpdateCompanyRequest
```

Controllerに届く前に、入力値が正しいかチェックできる。

---

### `$request->validated()`

`$request->validated()` は、FormRequestでバリデーションを通過した値だけを取り出すメソッド。

```php
$data = $request->validated();
```

これにより、不正な値や想定外の値をDBに保存しにくくなる。

---

### `Rule::in()`

`Rule::in()` は、指定した選択肢の中に値が含まれているか確認するLaravelのバリデーションルール。

```php
Rule::in([
    '応募済み',
    '書類選考待ち',
    '内定',
    '落選',
])
```

これにより、statusに想定外の文字列が入ることを防げる。

---

### `in_array()`

`in_array()` は、PHPの組み込み関数。

配列の中に指定した値が含まれているか確認する。

```php
in_array($company->status, [
    '応募済み',
    '書類選考待ち',
    '面談日程調整中',
    '面談後返答待ち',
]);
```

これは日本語にすると、

```txt
この会社のstatusが、確認待ち対象の一覧に含まれているか
```

という意味。

---

### `Company::all()`

`Company::all()` は、companiesテーブルの全件を取得するEloquentメソッド。

```php
$companies = Company::all();
```

ログイン機能なしの段階では使える。

しかしログイン後は、他人の企業まで混ざるため危険。

---

### `Company::where(...)->get()`

`where()` は条件を指定するメソッド。

`get()` は条件に合うデータを複数件取得するメソッド。

```php
Company::where('user_id', Auth::id())->get();
```

これは、

```txt
ログイン中ユーザー本人の企業だけ取得する
```

という意味。

---

### `count()`

`count()` は件数を数えるメソッド。

```php
$companies->count();
```

Dashboardの応募総数や、面談予定数などで使う。

---

### `filter()`

`filter()` は、Collectionの中から条件に合うものだけを残すメソッド。

```php
$companies->filter(function ($company) {
    return $company->status === '面談予定';
});
```

Reactの `filter()` と考え方が近い。

---

### `where()`

Collectionの `where()` は、Collection内のデータから特定条件に合うものを抽出する。

```php
$companies->where('status', '内定')->count();
```

これは、

```txt
status が 内定 の会社だけ数える
```

という意味。

---

### `sortBy()`

`sortBy()` は昇順に並び替えるCollectionメソッド。

```php
$interviewCompanies->sortBy('interview_date');
```

面談日が近い順に並べたいときに使う。

---

### `sortByDesc()`

`sortByDesc()` は降順に並び替えるCollectionメソッド。

```php
$highPriorityCompanies->sortByDesc('priority');
```

priorityが高い順に並べたいときに使う。

---

### `take()`

`take()` は先頭から指定件数だけ取得するCollectionメソッド。

```php
$companies->take(3);
```

ActionListsで上位3件だけ返すときに使う。

---

### `values()`

`values()` はCollectionの配列番号を振り直すメソッド。

```php
$companies->take(3)->values();
```

filterやsort後はキーが飛び飛びになることがあるため、JSONとして返す前に整える。

---

### `Auth::id()`

`Auth::id()` は、ログイン中ユーザーのIDを取得するLaravelのFacadeメソッド。

```php
use Illuminate\Support\Facades\Auth;

Auth::id();
```

JobHunt Liteでは、自分の企業だけ取得・登録・更新・削除するために使う。

---

### `abort_unless()`

`abort_unless()` は、条件を満たさない場合に処理を中断してHTTPエラーを返すLaravelの関数。

```php
abort_unless($company->user_id === Auth::id(), 403);
```

これは、

```txt
この企業が自分のものでなければ403で止める
```

という意味。

---

### `fresh()`

`fresh()` は、DBから最新状態のモデルを取り直すEloquentメソッド。

```php
return new CompanyResource($company->fresh());
```

保存後の最新データをResourceに渡すために使う。

---

### `refresh()`

`refresh()` は、現在のモデルインスタンス自体をDBの最新状態で更新するメソッド。

```php
$company->refresh();
```

`fresh()` は新しいインスタンスを返し、`refresh()` は今のインスタンスを更新する。

---

### `fillable`

`fillable` は、`create()` や `update()` で保存を許可するカラム一覧。

```php
protected $fillable = [
    'name',
    'status',
    'is_favorite',
];
```

ここに入っていない項目は、保存されないことがある。

---

### `casts`

`casts` は、DBの値をLaravel上で扱いやすい型に変換する設定。

```php
protected $casts = [
    'is_favorite' => 'boolean',
];
```

`0 / 1` を `false / true` として扱えるようにする。

---

### `foreignId()`

`foreignId()` は、外部キー用のIDカラムを作るLaravelのmigrationメソッド。

```php
$table->foreignId('user_id');
```

companiesテーブルとusersテーブルを紐づけるために使う。

---

### `constrained()`

`constrained()` は、外部キー制約を付けるメソッド。

```php
$table->foreignId('user_id')->constrained();
```

通常、`user_id` なら `users.id` を参照する。

---

### `cascadeOnDelete()`

`cascadeOnDelete()` は、親データが削除されたとき、関連する子データも削除する設定。

```php
$table->foreignId('user_id')
    ->constrained()
    ->cascadeOnDelete();
```

ユーザーが削除されたら、そのユーザーの企業データも削除される。

---

### `nullable()`

`nullable()` は、そのカラムに `null` を許可するmigrationメソッド。

```php
$table->foreignId('user_id')->nullable();
```

既存データがある状態でカラムを追加するときに使うことがある。

---

### `after()`

`after()` は、カラムをどのカラムの後ろに追加するか指定するmigrationメソッド。

```php
$table->boolean('is_favorite')->after('rejection_stage');
```

DB上のカラム順を見やすくするために使う。

---

## React / TypeScript側

### `useState()`

`useState()` は、Reactで状態を持つためのHook。

```tsx
const [isExpanded, setIsExpanded] = useState(false);
```

企業一覧の「もっと見る」や、ActionListsの開閉で使う。

---

### `props`

`props` は、親コンポーネントから子コンポーネントへ値や関数を渡す仕組み。

```tsx
<CompanyTable companies={companies} onToggleFavorite={toggleCompanyFavorite} />
```

子側ではpropsとして受け取る。

---

### `Partial<T>`

`Partial<T>` は、TypeScriptの型ユーティリティ。

すべてのプロパティを任意にする。

```tsx
function updateDetailForm(values: Partial<CompanyForm>) {
  setDetailForm((prev) => ({
    ...prev,
    ...values,
  }));
}
```

フォームの一部だけ更新したいときに使う。

---

### スプレッド構文 `...`

スプレッド構文は、既存のオブジェクトを展開する書き方。

```tsx
{
  ...prev,
  status: "落選",
}
```

これは、

```txt
既存の値を残しつつ、statusだけ上書きする
```

という意味。

---

### 関数型state更新

Reactのstate更新では、前回のstateを使って更新する場合、関数型更新が安全。

```tsx
setDetailForm((prev) => ({
  ...prev,
  status: "落選",
}));
```

古いstateを掴む問題を避けやすい。

---

### `fetch()`

`fetch()` は、FrontendからAPIを叩くブラウザ標準関数。

```tsx
const response = await fetch(`${API_BASE_URL}/companies`);
```

Laravel APIから企業一覧を取得するときに使う。

---

### `response.json()`

`response.json()` は、APIレスポンスをJSONとして読み取るメソッド。

```tsx
const result = await response.json();
```

APIがHTMLエラーを返していると、`Unexpected token '<'` になる。

---

### `localStorage`

`localStorage` は、ブラウザにデータを保存する仕組み。

```tsx
localStorage.setItem("jobhunt_token", data.token);
```

JobHunt Liteでは、ログイン後のtoken保存に使う。

---

### `Authorization: Bearer`

APIトークン認証で使うHTTPヘッダー。

```tsx
Authorization: `Bearer ${authToken}`;
```

Sanctumで守られたAPIにアクセスするために必要。

---

### `createHeaders()`

API通信のheadersを共通化するための関数。

```tsx
function createHeaders(token: string | null, hasBody = false) {
  return {
    ...(hasBody ? { "Content-Type": "application/json" } : {}),
    Accept: "application/json",
    ...(token ? { Authorization: `Bearer ${token}` } : {}),
  };
}
```

GET / POST / PUT / PATCH / DELETE で共通利用する。

---

### 条件演算子

条件によって値を切り替えるJavaScript / TypeScriptの書き方。

```tsx
company.isFavorite ? "赤ハート" : "白抜きハート";
```

いいね済みかどうかで表示を変えるときに使う。

---

### SVGの `fill`

SVGの `fill` は、図形の塗りつぶしを指定する属性。

```tsx
fill={company.isFavorite ? "currentColor" : "none"}
```

いいね済みなら塗りつぶし、未いいねなら白抜きにする。

---

### `currentColor`

`currentColor` は、CSSの文字色をSVGにも反映する値。

```tsx
text - red - 500;
fill = "currentColor";
```

この組み合わせで、赤いハートを作れる。

```php
$response = $this->getJson('/api/companies');
```

---

### `postJson()`

JSON APIにPOSTリクエストを送るテスト用メソッド。

```php
$response = $this->postJson('/api/companies', $data);
```

---

### `putJson()`

JSON APIにPUTリクエストを送るテスト用メソッド。

```php
$response = $this->putJson("/api/companies/{$company->id}", $data);
```

---

### `assertJsonPath()`

JSONレスポンスの特定の場所の値を確認するメソッド。

```php
$response->assertJsonPath('summary.total', 3);
```

---

### `assertJsonCount()`

JSON配列の件数を確認するメソッド。

```php
$response->assertJsonCount(1, 'actionLists.highPriority');
```

---

### `assertJsonMissing()`

指定したJSON断片がレスポンス全体に存在しないことを確認するメソッド。

注意点として、レスポンス全体を見るため、特定配列内だけを確認したい場合には向かないことがある。
