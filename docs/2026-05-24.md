# 8. 明日の最初の一手

## まずやること

routes/api.php と CompanyDashboardController を作る。

コマンド。

php artisan make:controller Api/CompanyDashboardController

次に route 追加。

use App\Http\Controllers\Api\CompanyDashboardController;

Route::get('/companies/dashboard', [CompanyDashboardController::class, 'index']);

次に最小実装。

public function index()
{
$companies = Company::all();

    return response()->json([
        'summary' => [
            'total' => $companies->count(),
            'interview' => $companies->where('status', '面談予定')->count(),
            'waiting' => $companies->whereIn('status', [
                '応募済み',
                '書類選考待ち',
                '面談日程調整中',
                '面談後返答待ち',
            ])->count(),
            'offer' => $companies->where('status', '内定')->count(),
            'rejected' => $companies->where('status', '落選')->count(),
            'highPriority' => $companies->filter(
                fn ($company) => (float) $company->priority >= 4.0
            )->count(),
        ],
    ]);

}

この後に、Feature Test を追加する。

# 9. 設計書 Excel に書く内容

## Dashboard API 設計

### 概要

JobHunt の Dashboard 表示に必要な集計情報を Laravel 側で生成する API。

### 目的

React 側で companies を何度も filter するのではなく、Dashboard 用データを API として返すことで、画面表示と集計ロジックの責務を分ける。

### エンドポイント

GET /api/companies/dashboard

### レスポンス項目

- summary
  - total
  - interview
  - waiting
  - offer
  - rejected
  - highPriority
- actionLists
  - interviews
  - waiting
  - highPriority
- alerts
  - type
  - message

### 表示条件

- interview
  - status = 面談予定
  - または interview_date が存在する
- waiting
  - status = 応募済み
  - status = 書類選考待ち
  - status = 面談日程調整中
  - status = 面談後返答待ち
- highPriority
  - priority >= 4.0
  - status が落選・辞退ではない

### テスト観点

- summary 件数が正しい
- 面談予定企業が返る
- 確認待ち企業が返る
- 高優先度企業が返る
- 終了状態の企業が除外される
