# JobHunt 画面設計

## 概要

JobHunt は、転職活動中の応募企業・選考状況・面談予定・次アクションを管理する Web アプリです。

MVP では、画面数を増やしすぎず、以下の 3 画面を中心に構成する。

- Dashboard
- CompanyList
- CompanyDetail / CompanyForm

## 画面一覧

| 画面名        | URL 案                  | 目的                                |
| ------------- | ----------------------- | ----------------------------------- |
| Dashboard     | `/` または `/dashboard` | 現在の転職活動状況を 5 秒で把握する |
| CompanyList   | `/companies`            | 応募企業を Excel 風に一覧管理する   |
| CompanyDetail | `/companies/:id`        | 1 社ごとの詳細情報を確認する        |
| CompanyCreate | `/companies/new`        | 企業情報を新規登録する              |
| CompanyEdit   | `/companies/:id/edit`   | 企業情報を編集する                  |

---

## 1. Dashboard

### 目的

現在の転職活動状況をすぐに把握する。

### 表示項目

- 応募総数
- 面談予定数
- 返答待ち件数
- 内定数
- 落選数
- 今日の面談
- 今週の面談
- 次アクション一覧
- 選考ファネル

### 表示イメージ

```txt
応募総数：50
面談予定：3
返答待ち：5
内定：1
落選：31

今日の予定
- 14:00 株式会社〇〇 1次面接

次アクション
- 株式会社A：面談日程返信
- 株式会社B：職務経歴書送付
```

### 優先度

高

---

## 2. CompanyList

### 目的

応募企業を一覧で管理する。

Excel の転職ログに近いメイン画面として扱う。

### 表示項目

- 企業名
- 媒体
- 状況
- 応募日
- 面談日
- 次アクション
- 書類選考
- 1 次面接
- 2 次面接
- 最終結果
- 落選段階
- メモ

### 機能

- 企業一覧表示
- 企業名検索
- 媒体フィルター
- 状況フィルター
- 登録画面への遷移
- 詳細画面への遷移
- 編集
- 削除

### 表示イメージ

```txt
企業名        媒体      状況              応募日      面談日
株式会社A    type     面談予定          5/17       5/23 14:00
株式会社B    Green    書類選考待ち      5/18       -
株式会社C    レバテック 落選            5/18       -
```

### 優先度

高

---

## 3. CompanyDetail

### 目的

1 社ごとの選考状況、URL、メモ、次アクションを確認する。

### 表示項目

- 企業名
- 媒体
- 状況
- 応募日
- 面談日
- 求人 URL
- 面談 URL
- メモ
- 次アクション
- 書類選考結果
- 1 次面接結果
- 2 次面接結果
- 最終結果
- 落選段階
- Google カレンダー追加ボタン

### 機能

- 詳細表示
- 編集画面への遷移
- 削除
- Google カレンダー登録 URL 生成

### 優先度

高

---

## 4. CompanyCreate / CompanyEdit

### 目的

企業情報を登録・編集する。

### 入力項目

- 企業名
- 媒体
- 状況
- 応募日
- 面談日
- 求人 URL
- 面談 URL
- メモ
- 次アクション
- 書類選考結果
- 1 次面接結果
- 2 次面接結果
- 最終結果
- 落選段階

### バリデーション案

| 項目         | 必須 | 内容                       |
| ------------ | ---- | -------------------------- |
| 企業名       | 必須 | 255 文字以内               |
| 媒体         | 任意 | 100 文字以内               |
| 状況         | 必須 | 定義済みステータスから選択 |
| 応募日       | 任意 | 日付                       |
| 面談日       | 任意 | 日時                       |
| 求人 URL     | 任意 | URL 形式                   |
| 面談 URL     | 任意 | URL 形式                   |
| メモ         | 任意 | 長文                       |
| 次アクション | 任意 | 255 文字以内               |

### 優先度

高

---

## MVP での画面方針

最初から画面を増やしすぎない。

優先順位は以下。

1. CompanyList
2. CompanyCreate / CompanyEdit
3. CompanyDetail
4. Dashboard

まずは企業 CRUD を完成させ、その後 Dashboard で集計表示する。
