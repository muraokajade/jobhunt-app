# JobHunt コンセプト設計

## 概要

JobHunt は、転職活動中の応募企業・選考状況・面談予定・次アクションを一元管理する転職活動 CRM です。

## 目的

複数の求人媒体・エージェント・企業応募が並行する転職活動において、現在の選考状況と次にやるべき行動をすぐに把握できるようにする。

## ゴール

5 秒で、今の選考状況と次にやることがわかる状態を作る。

## 背景

実際の転職活動では、Excel で応募企業、媒体、状況、応募日、面談日、メモ、次アクション、選考結果を管理していた。

Notion よりも Excel 形式の方が一覧性が高く、応募状況を素早く把握しやすかった。

その実体験をもとに、Excel の速さと Web アプリの拡張性を組み合わせた転職活動管理アプリとして JobHunt を作成する。

## 想定ユーザー

- 転職活動中の人
- 複数企業に応募している人
- Green、type、レバテック、Wantedly、エージェントなど複数媒体を使っている人
- 面談日や次アクションが混乱しやすい人
- Excel や Notion で管理しているが、もっと速く状況を見たい人

## コア価値

- 応募状況が一目でわかる
- 面談予定がすぐわかる
- 次にやることが明確になる
- 選考ファネルで現在地が見える
- 転職活動中の焦りを減らせる

## MVP でやること

- 企業一覧管理
- 企業登録
- 企業編集
- 企業削除
- 選考ステータス管理
- 応募日管理
- 面談日管理
- 次アクション管理
- メモ管理
- 落選段階管理
- 選考ファネル可視化
- Google カレンダー登録 URL 生成

## MVP でやらないこと

- 求人検索
- 履歴書作成
- 職務経歴書作成
- AI 志望動機生成
- Google OAuth 連携
- エージェント連携
- チーム共有
- 複雑な通知
- ガントチャート

## 開発目的

- Laravel / React / TypeScript の実務感覚を戻す
- 実体験ベースの課題を Web アプリ化する
- リベンリ参画前に実務的な CRUD・API 設計・画面設計を復習する
- 将来的に販売可能な Web ブラウザアプリへ展開する
