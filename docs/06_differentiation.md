# JobHunt 差別化設計

## 目的

既存の Job Application Tracker 系アプリとの差別化ポイントを整理する。

JobHunt は単なる応募記録アプリではなく、日本の転職活動に特化した高速 CRM として設計する。

## 既存アプリの傾向

既存の求人応募管理アプリには、以下の機能が多い。

- 会社名管理
- 応募ステータス管理
- 求人 URL 管理
- 応募日管理
- メモ管理
- フォローアップ管理
- 面談予定管理
- 統計・チャート表示

そのため、単なる CRUD だけでは差別化しづらい。

## JobHunt の差別化方針

### 1. 日本の転職活動フローに特化する

海外アプリのような大雑把なステータスではなく、日本の転職活動で実際に使う状態を管理する。

```txt
応募済み
書類選考待ち
書類通過
面談日程調整中
面談予定
面談後返答待ち
内定
辞退
落選
```
